## what
- ソースコードを書くときと同じツールを使ってドキュメントを書くという手法、またはその考え方や哲学のこと。
- 4原則
	- markdownなど軽量マークアップ言語で記述する
	- ソースコードと同じリポジトリに履歴管理する
	- レビューを通した承認フローを経る
	- マージされると自動でリントが実行され、静的サイトとして自動公開される。
## why
- 従来の Excel や Word、Wiki を使ったドキュメントライティングでは、ソースコードと保存場所が異なるために、コードとドキュメントの内容が乖離してしまうという問題があります。そこで、ソースコードとドキュメントを同じリポジトリ内に収め、かつ同じツールを使って書くことによって、その問題を解決する。
