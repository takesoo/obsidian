## what
- ソフトウェアアーキテクチャーを4つの図によって表現・解説するための図解方法。順に抽象から具体へとズームアップする構造になっており、全体像から詳細が理解しやすいようになっている。
	- system context
	- container
	- compornent
	- cod
### system context
最も全体像を説明するための図解。対象となるソフトウェアシステムが、その周囲の世界の中でどのような位置づけにあるのかを示す。
![[Pasted image 20260714214243.png]]
### container
対象となるソフトウェアシステムを拡大表示し、その内部にあるアプリケーションとデータストアを示す。
![[Pasted image 20260714214258.png]]
### compornent
個々のコンテナを拡大表示し、その内部にあるコンポーネントを示す。
![[Pasted image 20260714214311.png]]
### code
ここのコンポーネントをさらに拡大表示し、そのコンポーネントがコードレベルでどのように実装されているかを示す（UMLのクラス図など）。
![[Pasted image 20260714214324.png]]

## why
- 既存の[[Unified Modeling Language|UML]]などのツールは、学習コストが大きく、正しく使われていない。一方で、多くの開発チームは、よりシンプルなボックスと線による図をホワイトボードに書き殴るが、これでは説明力が足りないために、開発チーム内での視覚的な図によるコミュニケーション能力が失われている。
