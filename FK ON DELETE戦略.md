## what
- 外部キー制約で参照先（親）の行が削除されたときに参照元（子）の行をどう扱うか

| 戦略            | 動作                                   | 用途例                  |
| ------------- | ------------------------------------ | -------------------- |
| `CASCADE`     | 親が削除されたら子も自動削除                       | 注文明細（注文が消えれば明細も不要）   |
| `SET NULL`    | 親が削除されたら子のFK列を `NULL` にする            | 任意参照（担当者が退職しても記録は残す） |
| `SET DEFAULT` | 親が削除されたら子のFK列をデフォルト値にする              | 稀に使用                 |
| `RESTRICT`    | 子が存在する限り親の削除を禁止                      | 誤削除防止（デフォルトに近い挙動）    |
| `NO ACTION`   | トランザクション終了時点で整合性チェック（`RESTRICT` に近い） | PostgreSQL のデフォルト    |
